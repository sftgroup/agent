# OpenClaw Instance Doctor — 人话说明

## 这玩意儿干嘛的？

你有 7 台 OpenClaw 服务器。每台都有一堆配置项。时间长了，有的配了有的没配、有的改了这个没改那个——**配置漂移**。

这个工具就是**一键体检 + 一键修复**：
1. 连上任意一台服务器
2. 跑个脚本，告诉你哪儿不对
3. 再跑个脚本，自动修好

## 为什么需要它？

OpenClaw 的快慢取决于**缓存命中率**。缓存命中了，响应秒出；缓存没命中，要多花几秒和几百个 token 重新建缓存。

想让缓存稳定命中，需要 7 个配置项全部对齐：

```
1. 前缀稳定（每次都发一样的开头）
2. 缓存不丢（1 小时内不过期）
3. 保活心跳（快过期时续一把）
4. 瘦身（旧的无用内容清掉）
5. 空间够（文件能完整加载）
6. 不报 404（跳过多余文件）
7. 压缩合理（窗口比例合适）
```

这 7 项有一个不对，缓存命中率就掉。

## 效果

| 优化前 | 优化后 |
|:------:|:------:|
| 缓存命中 ~50% | 缓存命中 **82%** |
| 每次交互多花 200+ token | 省掉重复缓存开销 |
| 响应时快时慢 | 稳定快速 |

team3 已经验证了，缓存命中率从 50% 提升到 82%。

## 怎么用

### 第一步：体检

```bash
sshpass -p 'Asdf1234!' ssh ubuntu@目标IP 'bash -s' < diagnose.sh
```

输出 8 个维度的报告：

| 维度 | 看什么 |
|------|--------|
| 基础信息 | 谁、哪、磁盘多大、跑多久了 |
| 主配置 | AGENTS.md 在不在、行数对不对 |
| 子 agent | 9 个 agent 齐不齐 |
| 旧版残留 | 有没有废弃目录 |
| Config | 15 项配置逐行对比 |
| **缓存链** | **7/7 评分卡** ← 最核心 |
| 浏览器 | 清理脚本配了没 |
| 磁盘 | 哪个目录吃空间 |

### 第二步：修复

```bash
# 先停 Gateway
ssh ubuntu@目标IP "openclaw gateway stop"

# 一键修复（11 项检查，自动对齐）
sshpass -p 'Asdf1234!' ssh ubuntu@目标IP 'bash -s' < fix-config.sh

# 重启
ssh ubuntu@目标IP "openclaw gateway start"
```

### 第三步：验证

再跑一次体检，确认缓存链 7/7。

## 修了哪些东西？

| 修什么 | 为什么重要 |
|--------|-----------|
| 缓存保留策略 | 不设的话，每次请求缓存从头建 |
| 心跳保活 | 没心跳，缓存过期后就丢了 |
| 旧内容清理 | 上下文里堆太多无用的旧工具输出 |
| 文件加载上限 | 10 个 agent 的配置要能完整加载 |
| 跳过多余文件 | 少了 404 报错，前缀更稳定 |
| 压缩参数 | 压缩太激进会丢上下文，太保守会爆窗口 |
| 会话管理 | 旧 session 堆积多了占磁盘 |

## 安全提示

- 所有操作通过 SSH 执行，不修改代码
- 修复脚本只动 `openclaw.json` 的配置部分
- 不会删 `agents/` 目录（那是运行时数据）
- 磁盘清理脚本只清包管理器缓存，不碰项目文件
